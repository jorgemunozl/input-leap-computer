## ÔøΩ Why Isn't My Server Starting Automatically?

**Input Leap servers are NEVER automatic!** Here's why:

1. **Security**: Servers don't auto-start to prevent unauthorized access
2. **Control**: You decide when to share your mouse/keyboard
3. **Resources**: Only runs when you actually need it

**The workflow is:**
1. **Main computer**: `leap-server start` (NEW!) or GUI ‚Üí Click "Start"
2. **This machine**: `leap-client connect` (NEW!) or `leap start`
3. **Use Input Leap**: Move mouse/keyboard seamlessly
4. **Done**: `leap-server stop` or stop server GUI when finished

**Common mistakes:**
- ‚ùå Expecting server to start automatically 
- ‚ùå Running `leap start` on server (use `leap-server start` instead!)
- ‚ùå Forgetting to start server before connecting client
- ‚ùå Not configuring client IP on server
- ‚úÖ `leap-server start` first, then `leap-client connect`

- **üíæ Persistent config** - Remembers your settings across reboots
- **üîß Manual override** - Set custom IPs for any interface
- **üìã Backup safety** - Saves old config before changes

**üîß Ethernet Command Options:**

- **`leap ethernet forever`** - Permanent setup (systemd-networkd, survives reboots) ‚ö° **RECOMMENDED**

**üí° When to use each:**

**Smart Network Features:**
- **üîç Auto-detects Ethernet** - Prioritizes wired over WiFi
